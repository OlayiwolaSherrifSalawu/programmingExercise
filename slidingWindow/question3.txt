Challenge 002: The Anagram DetectiveThe Scenario:You are building a search engine feature that needs to find exact anagrams of a target word hidden inside a massive document.An anagram is a word formed by rearranging the letters of a different word, using all the original letters exactly once. (e.g., "cba" is an anagram of "abc").The Data Structure & Rules:You need to find the starting index of every valid anagram of the target word inside the long string.You must do this in a single pass ($O(N)$ time).Expected Input:document: A long string of random lowercase letters (e.g., "cbaebabacd").target: The specific word you are searching for (e.g., "abc").Expected Output:A list of integers representing the starting indices of where the anagrams begin.Example Output: [0, 6](Explanation: The substring from index 0 is "cba", which is an anagram of "abc". The substring from index 6 is "bac", which is an anagram of "abc".)
